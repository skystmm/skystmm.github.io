养龙虾——让 AI 真正认识你

你的龙虾还在冬眠吗？

先聊聊：什么是养龙虾

“龙虾”是 OpenClaw 的昵称。

OpenClaw 是一个开源的个人 AI Agent 框架，2026 年 1 月在技术圈爆红。

它不是 ChatGPT 那样的聊天工具。

它是一个”住在电脑里的 AI 管家”——能操作你的文件、运行命令、浏览网页、连接你的飞书和 Telegram。

一句话概括：

它标志着 AI 从”对话工具”向”自主执行者”的范式转变。

怎么装

前置要求

Node 22 或更新版本。

检查一下：node --version

安装命令

根据你的系统选择：

系统 命令
macOS / Linux curl -fsSL https://openclaw.ai/install.sh | bash
Windows (PowerShell) iwr -useb https://openclaw.ai/install.ps1 | iex

装完之后

运行 onboarding 向导：

它会引导你配置模型、通道等。

检查 Gateway 是否运行：

打开 Control UI：

懒人方案：云端一键部署

不想折腾命令行？

腾讯云、百度智能云都有 OpenClaw 应用模板。

买个 2 核 2G 的云服务器，一键部署，省心省力。

一个月二三十块。

重要提醒：

不要在你的主电脑上部署。

OpenClaw 权限很高，安全起见，用云服务器或闲置主机。

装好之后，大多数人会遇到同一个问题——

装完就落灰，只会聊天。

它不是你装完就能一劳永逸的工具，而是要你慢慢磨合、慢慢培养的打工仔。

养龙虾的第一步，不是让它干活。

是让它认识你。

岗前培训：告诉龙虾你是谁

新员工入职，要填档案。

龙虾也一样。

不知道你是谁，它就只能给你通用回复——没有针对性，没有个性，没有灵魂。

两份档案：SOUL.md 和 USER.md

OpenClaw 用两个文件来定义”身份”：

SOUL.md——定义 AI 的灵魂

它的价值观、行为准则、性格特点。

你想让它是什么性格？严谨还是活泼？正式还是随意？

有什么绝对不能做的事？有什么必须遵守的原则？

USER.md——定义用户的信息

你是谁？做什么的？有什么偏好？

这是让它”认识你”的档案。

先填 SOUL.md：塑造你的 AI

打开工作目录，找到 SOUL.md。

写下你想让它成为什么样的 AI。

一个示例：

这个文件决定了 AI 的”性格”。

认真写。

再填 USER.md：让它认识你

同一个目录下，找到 USER.md。

填写三个维度：

身份信息

你是谁？做什么的？在哪个城市？

工作偏好

几点上班？几点睡觉？

沟通风格是简洁还是详细？

生活习惯

健身吗？学习吗？阅读吗？

一个示例：

两份档案的区别

文件 定义什么 谁来填
SOUL.md AI 的性格、价值观、行为准则 你决定它成为谁
USER.md 你的身份、偏好、习惯 让它认识你

还有哪些档案

OpenClaw 还会用到这些文件：

文件 作用
AGENTS.md AI 每次醒来先读的工作指南，定义工作流程和规则
IDENTITY.md AI 的”身份证”，名字、emoji、头像
TOOLS.md 本地配置笔记，SSH 主机、设备名称等环境信息
HEARTBEAT.md 定时任务清单，让 AI 定期检查的事项
BOOTSTRAP.md 首次运行的引导脚本，完成后可删除

填完 SOUL.md 和 USER.md，你的龙虾就已经认识你了。

其他文件可以慢慢补充。

技能培训：给龙虾装超能力

为什么需要技能

龙虾出厂配置：只会聊天。

这就像一个只会说话的员工——

能聊，但干不了活。

想让 OpenClaw 真正成为”打工仔”，得给它装技能。

技能是什么？

就是让 AI 学会新能力的”插件包”。

装了 Summarize，它会摘要长文。

装了 Agent Browser，它能操作浏览器。

装了 GitHub 技能，它能帮你管代码仓库。

位置 说明
~/.openclaw/skills/ 所有 Agent 共享的技能
<workspace>/skills/ 当前工作区专属技能

必装技能清单

以下 7 个技能，建议优先安装：

  • Capability Evolver — 越用越顺手，自动学习你的偏好
  • Self-Improving Agent — 犯错能改，从错误中学习进化
  • Summarize — 长文秒懂，文档摘要神器
  • Find Skills — 发现更多技能，技能导航
  • Agent Browser — 解锁浏览器，自动化网页操作
  • Skills-Vetter — 安全审核，安装技能前先检查风险
  • 场景刚需技能 — 飞书、GitHub、Weather，按需安装

安全提示

安装技能前，先用 Skills-Vetter 审核。

技能本质上是代码，可能有安全风险。

来自陌生人的技能，就像陌生人的软件——先检查，再安装。

检查要点：

  • 是否请求敏感权限？
  • 是否有网络请求？
  • 代码是否开源可审计？

光把这几件事搞定，你的龙虾就已经超过 90% 的新手了。

组队搭档：龙虾的最佳拍档

龙虾单独用已经很强。

但配合其他工具，效果翻倍。

搭档 场景 产出
Claude Code 写代码 最强编程搭档
n8n 自动化 流程自动跑
Obsidian 知识库 第二大脑
飞书/Telegram 通讯 随身助理

怎么选？

看你最常在哪个场景工作。

程序员优先装 Claude Code。

运营优先接飞书。

飞书接入步骤

想把 OpenClaw 接入飞书？按这个流程来：

第一步：创建飞书应用

访问 飞书开放平台，创建应用。

获取 App ID 和 App Secret。

第二步：配置权限

至少开通这些权限：

  • im:message — 发送消息
  • im:message:send_as_bot — 以机器人身份发送
  • im:resource — 上传资源

第三步：添加机器人能力

在应用管理中，添加”机器人”能力。

配置事件订阅。

第四步：运行 OpenClaw 配置向导

选择飞书通道，填入 App ID 和 App Secret。

详细文档：https://docs.openclaw.ai/zh-CN/channels/feishu

配置完成后，你就可以在飞书里直接和 OpenClaw 对话了。
